Game Project

FULL PAGE

I created a simple game called CoinQuest using GDevelop, a beginner-friendly game development platform that helped me understand the fundamentals of coding logic. GDevelop uses an event-based system instead of traditional text code, which made it easier for me to learn how conditions and actions work in games. Through this platform, I learned important basic concepts such as player movement, collision detection, object interactions, and simple game rules without needing to write complex code right away.

Starting CoinQuest from scratch allowed me to apply what I learned step by step. I had to think logically about what should happen when the player collects a coin or touches a monster, which helped me develop problem-solving skills similar to real programming. Even though GDevelop is a visual platform, it taught me how to structure game logic, debug errors, and understand how different parts of a program connect. This experience helped me build the mindset needed for actual coding, such as breaking tasks into smaller steps and testing ideas continuously.

Working with GDevelop helped me build a strong foundation for learning real programming languages in the future. The concepts I learned—like conditions, variables, events, and game flow—are also used in languages such as JavaScript, Python, and C++. As I continue improving CoinQuest, I can add more advanced mechanics, better enemy behavior, and improved game design. Over time, these improvements will help me transition from visual coding to writing real code, making this project an important starting point in my journey as a programmer and game developer.
