I created a very simple game called CoinQuest, where the main goal is to collect coins while avoiding monsters. The game focuses on basic challenges and quick reflexes, making it easy to play but still fun. Despite its simplicity, CoinQuest allowed me to experiment with game mechanics like movement, collision detection, and scoring, giving me a hands-on experience in creating interactive digital content.

I built this game from scratch using GDevelop, a beginner-friendly platform that allowed me to focus on coding concepts without being overwhelmed by complex programming languages. Through the process, I learned how to set up scenes, create player controls, spawn objects, and implement basic game logic. Working on CoinQuest taught me how to think logically, break problems into smaller steps, and troubleshoot issues when something didnâ€™t work as expected.

Creating CoinQuest showed me how even a very simple idea can turn into a playable game with enough patience and practice. I realized that game development is an iterative process where learning and experimenting are key. In the future, I can improve CoinQuest by adding more levels, different types of monsters, power-ups, or even sound effects to make it more engaging. This project has sparked my interest in coding and game design, and I am excited to continue learning and making more complex games over time.
